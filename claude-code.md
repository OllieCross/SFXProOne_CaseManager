# Prompt

i have been assigned a coding project to create a simple full-stack web app for a rental company where:

- A stagehand can log in with username/password
- A stagehand can scan a qr code on a flight case through the website with his phone camera
- After scanning the QR code he/she gets a full list of gear that should be put in that case, zero or more images of the inside of the case, and also zero or more available PDFs for manuals and certificates that he can view directly in the browser
- There could also be cases where the QR code only contains PDFs without a list of gear (for example when a sticker is put on a machine instead of a case, where a machine does not have a list of gear but it needs a PDF manual and a PDF certificate)
- There should be three roles advertised based on login: viewer, editor and admin
- Viewer can only scan QR codes and read the contents and PDFs
- Editor can upload new cases with QR codes, edit the contents of the existing ones and upload new PDF documents and Images
- Admin role is the same as editor with extra future features like logging, history, user management etc.
- The Website needs to be able to handle various types of image media including .jpg, .jpeg, .png, .webp and other used image file extensions
- .HEIC images that are popular on modern iOS devices will be converted to a .jpg on a client-side using heic2any
- resize/compress the images, maximum width/length 1920px, compressed to quality 0.8, using a library like browser-image-compression
- The Editor should be able to choose if he/she wants to upload a photo from a phone (file permissions) or take a photo directly in the browser
- The Editor should also be able to move items from one case to another
- The project should support Direct Uploads of files via Pre-signed URLs directly to MinIO without bottlenecking next.js
- The Project needs to be able to handle legacy QR codes that have already been created and put on the cases in the past using Google Keep with links like <https://keep.google.com/u/0/#NOTE/xyz123456789>
- The Project also needs to be able to handle any type of QR code. The plan is to hand out the editor a sheet of QR code stickers (not generated by this App) for new cases, where the QR code contains only a long string of numbers and letters, not full URLs in the event of Domain migration or provider change. (e.g. i dont want the QR code stickers to contain <https://sfxproone.olliecross.com/case/xyz1234> if the CEO decides to change the IT administrator or if i change/sell my olliecross.com domain)
- The finished project is being aware of the security needed for blocking possible SQL injections, brute-force attacks, enumerations and other type of attacks that might occur
- Use HTTPS with Traefik and Letsencrypt
- Both iOS and Apple users (stagehands) are going to use a home screen chrome bookmark that leads to the home page
- The Project needs to be able to keep login data for at least half an hour between inactivity (imagine a stagehand packing various cases on the stage, he doesnt have time to login everytime he opens the app)
- Frontend needs to be readable on both desktop and phone (phone is priority)
- The app doesnt need offline support
- Project will be deployed in docker containers via Docker Compose
- Always try to build and deploy custom Dockerimages using multiple build stages if possible in order to save disk space, shorten deploy time and tighten the security of the whole project

## Technology Stack

- Frontend : Next.js 14+ (App Router)
- Styling : Tailwind CSS
- Language : TypeScript
- QR Scanning : html5-qrcode
- PDF Viewing : PDF.js (Mozilla)
- Backend : Next.js API Routes / Node.js
- ORM : Prisma DB
- Database : PostgreSQL 18
- Auth : NextAuth.js v5 + Authentik OIDC/OAuth2
- Reverse Proxy : Traefik v3 + Letsencrypt SSL
- Containers : Docker + Docker Compose
- File Storage : MinIO (S3-compatible)

## Docker Compose Stack

- nextjs-app : frontend + API
- PostgreSQL : DB
- redis : Cache + Rate-limiting
- minIO : PDF/Img storage
